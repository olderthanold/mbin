Local LLM Setup (CPU-only)

Architecture:
LLM (llama.cpp) → HTTP API → Python (uv) → MCP (later)

0️⃣ Prerequisites

System packages

sudo apt update
sudo apt install -y \
  git \
  build-essential \
  cmake \
  python3

Install uv

curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.bashrc
uv --version

1️⃣ Get llama.cpp

cd ~
mkdir -p ai
cd ai
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp

2️⃣ Build (CPU optimized)

cmake -B build
cmake --build build -j
Output binaries:
build/bin/llama-server
build/bin/llama-cli

3️⃣ Download Model

Use GGUF format, Q4_K_M quantization

  • Phi-3 Mini (recommended)
  • Gemma 2B
  • TinyLlama

4️⃣ Run Server

./build/bin/llama-server \
  -m model.gguf \
  -t 4 \
  -c 2048 \
  --port 8080
Server: http://localhost:8080

5️⃣ Python Access (uv)

uv venv
uv pip install requests
import requests

r = requests.post(
    "http://localhost:8080/completion",
    json={"prompt": "Hello"}
)

print(r.json())

6️⃣ MCP Integration (later)

Python (uv)
   ↓
MCP server
   ↓
llama.cpp server

7️⃣ Tuning

  • -t → CPU threads
  • -c → context size (RAM)
  • Quantization → biggest impact

Summary

  • Minimal system deps
  • uv for Python
  • llama.cpp as engine
  • GGUF model
  • HTTP API ready
  • MCP-ready architecture
